# Gladiator Roster

Non-permanent roles spawned during tournament/deathmatch. Pick relevant gladiators per challenge.

| Gladiator | Fighting Style | Deploy When |
|-----------|---------------|-------------|
| @architect | "Here's how to make it work" | Always (default defender) |
| @cost-analyst | "What does this cost in money, time, opportunity?" | Infrastructure, build vs buy |
| @user-advocate | "What does the user actually experience?" | Decisions with UX consequences |
| @ops-gladiator | "Who deploys this at 2am when it breaks?" | New services, infra changes |
| @legacy-defender | "The current thing works. Prove the new thing is worth it." | Rewrite/migration proposals |
| @future-self | "2 years from now. Does this still make sense?" | Tech choices, dependency adoption |
| @simplicity | "Can you explain this to a new dev in 5 minutes?" | Over-engineering detection |

Don't force a gladiator into the arena if they have nothing meaningful to argue.
